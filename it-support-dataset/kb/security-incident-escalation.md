# Security Incident Escalation

**Article ID:** KB-021

## Summary
How to escalate a suspected security incident and severity definitions.

## Symptoms
- Suspected account compromise
- Malware alert on an endpoint
- Data exposure concern

## Troubleshooting Steps
1. Capture time, accounts, and systems involved.
2. Do not attempt remediation that destroys evidence.
3. Contain the affected account/device if instructed.

## Resolution Steps
1. Security triages, assigns severity, and drives containment/eradication.
2. Affected credentials reset; MFA re-enrolled if needed.
3. Post-incident review documented.

## Escalation Guidance
Regulatory-impacting incidents -> Compliance and senior Security leadership immediately.
